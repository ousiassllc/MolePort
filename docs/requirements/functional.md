# MolePort 機能要件

## 概要

MolePort は、SSH ポートフォワーディングを管理する Go 製ツールである。
バックグラウンドデーモンが SSH 接続とポートフォワーディングを永続管理し、CLI サブコマンドおよび TUI ダッシュボードから操作する。
`~/.ssh/config` からホスト情報を読み込み、ポート転送の設定・接続・切断を直感的に行える。

バイナリ名は `moleport` とする。

## ユースケース

### UC-1: デーモンの起動

- **アクター**: ユーザー
- **概要**: バックグラウンドデーモンを起動し、SSH 接続管理を開始する
- **基本フロー**:
  1. `moleport daemon start` を実行する
  2. 既存デーモンの稼働確認（PID ファイル + ソケット疎通）
  3. デーモンが存在しなければ、プロセスを自己フォークしてバックグラウンド化
  4. PID ファイル・Unix ソケットを作成
  5. config.yaml を読み込み、state.yaml からセッションを復元
  6. auto_connect ルールの自動接続を開始
  7. 「デーモンを起動しました (PID: xxxx)」と表示
- **代替フロー**:
  - デーモンが既に稼働中の場合: 「デーモンは既に稼働中です (PID: xxxx)」と表示

### UC-2: デーモンの停止

- **アクター**: ユーザー
- **概要**: バックグラウンドデーモンを停止する
- **基本フロー**:
  1. `moleport daemon stop` を実行する
  2. デーモンに shutdown リクエストを送信
  3. デーモンが全接続をグレースフルに切断し、状態を保存
  4. 「デーモンを停止しました」と表示
- **代替フロー**:
  - デーモンが稼働していない場合: 「デーモンは稼働していません」と表示

### UC-2a: デーモンの強制終了

- **アクター**: ユーザー
- **概要**: 応答しないデーモンを SIGKILL で強制終了する
- **前提条件**: `moleport daemon stop` がタイムアウトまたは応答しない
- **基本フロー**:
  1. `moleport daemon kill` を実行する
  2. PID ファイルからデーモンの PID を読み取る
  3. SIGKILL でプロセスを強制終了する
  4. PID ファイルを削除する
  5. 「デーモンを強制終了しました (PID: xxxx)」と表示
- **代替フロー**:
  - デーモンが稼働していない場合: 「デーモンは稼働していません」と表示
- **注意**: グレースフルな切断や状態保存は行われない

### UC-3: SSH ホスト一覧の表示

- **アクター**: ユーザー
- **概要**: `~/.ssh/config`（Include ディレクティブ含む）からホスト情報を読み込み、一覧表示する
- **CLI**: `moleport list`
- **TUI**: ダッシュボード上部に常時表示
- **表示情報**: ホスト名、HostName（実アドレス）、Port、User、接続状態、アクティブ転送数

### UC-4: SSH ホストへの接続

- **アクター**: ユーザー
- **概要**: 指定ホストに SSH 接続し、auto_connect ルールの転送を開始する
- **CLI**: `moleport connect <host>`
- **TUI**: TUI から SSH connect/disconnect は行えない。CLI の `moleport connect` / `moleport disconnect` を使用する
- **基本フロー**:
  1. デーモンに ssh.connect リクエストを送信
  2. デーモンが SSH 接続を確立
  3. 当該ホストの auto_connect ルールのフォワーディングを自動開始
  4. 結果を表示

### UC-5: ポートフォワーディングの追加

- **アクター**: ユーザー
- **概要**: 新しいポート転送ルールを追加する
- **CLI**: `moleport add --host <host> --type <type> --local-port <port> [--remote-host <host>] [--remote-port <port>] [--remote-bind-addr <addr>] [--name <name>] [--auto-connect]`
- **TUI**: SetupPanel でホストを選択し `Enter` キーでフォワード追加ウィザードを開始
- **基本フロー**:
  1. CLI フラグで対象ホスト（`--host`）・転送種別（`--type`: local/remote/dynamic）・ポート情報を指定する
  2. デーモンに forward.add リクエストを送信
  3. 設定が検証され、転送ルールが登録される
  4. 自動的にフォワーディングを開始する
- **代替フロー**:
  - ポート競合がある場合、エラーメッセージを表示し別のポートを提案する

### UC-6: ポートフォワーディングの開始・停止

- **アクター**: ユーザー
- **概要**: 登録済みの転送ルールを個別に開始・停止する
- **CLI**: `moleport start <name>` / `moleport stop <name>`
- **TUI**: 転送一覧で Enter キーでトグル、`d` キーで停止

### UC-7: 接続状態のリアルタイム監視

- **アクター**: ユーザー
- **概要**: アクティブな転送の状態をリアルタイムで監視する
- **CLI**: `moleport status`（スナップショット）
- **TUI**: ダッシュボードにリアルタイム表示（イベントサブスクリプション経由）
- **表示情報**:
  - 転送種別（Local / Remote / Dynamic）
  - ローカルポート ← → リモートホスト:ポート の視覚的表示
  - 接続状態（接続中 / 停止 / エラー / 再接続中）
  - 接続時間（稼働時間）
  - 転送データ量（送信/受信）
  - エラー状態の詳細
  - 再接続回数

### UC-8: 自動再接続

- **アクター**: システム（デーモン）
- **概要**: SSH 接続が切断された場合に自動でリトライし、フォワードルールを復元する
- **前提条件**: `reconnect.enabled` が `true`（デフォルト）
- **基本フロー**:
  1. デーモンが KeepAlive 応答途絶により接続切断を検知する
  2. SSH 接続状態を `Reconnecting` に更新し、`event.ssh` で通知する
  3. 当該ホストの全アクティブフォワードルールの状態を `SessionReconnecting` に更新し、`event.forward` で通知する
  4. ジッター付き指数バックオフでリトライする（初期 1s × (1 + 0〜10% ランダム)、倍増、最大 60s）
  5. SSH 再接続成功時:
     a. SSH 接続状態を `Connected` に更新し、`event.ssh` で通知する
     b. `SessionReconnecting` 状態の全フォワードルールを順次再開する
     c. 各ルールの再開成功時: 状態を `Active` に戻し、`ReconnectCount` をインクリメントする
     d. 各ルールの再開失敗時（ポート競合等）: 状態を `Error` にし、`event.forward` で通知する
  6. フォワード復元完了後、復元結果のサマリーをログ出力する
- **代替フロー**:
  - 最大リトライ回数に達した場合: SSH 状態を `Error`、全フォワード状態を `Error` にし通知する
  - 認証失敗（パスワード認証のみのホスト）: SSH 状態を `PendingAuth` にし、ユーザーの手動 `connect` を待つ（UC-14 と同様の挙動）
  - 再接続中にユーザーが `disconnect` を実行した場合: 再接続ループを中止し、`Disconnected` にする
  - デーモンがシャットダウン中の場合: 再接続ループを中止する
- **制限事項**:
  - パスワード認証・keyboard-interactive 認証のみのホストは、自動再接続時にクレデンシャルを提供できないため `PendingAuth` 状態になる。鍵認証または SSH エージェント利用が推奨される

### UC-9: セッション復元

- **アクター**: システム（デーモン）
- **概要**: デーモン起動時に前回のアクティブな転送を自動復元する
- **基本フロー**:
  1. デーモン起動時に `~/.config/moleport/state.yaml` から前回の状態を読み込む
  2. アクティブだった転送ルールを順次再接続する
- **補足**:
  - state.yaml からの復元と、config.yaml の `auto_connect` ルールの自動開始は独立した仕組みである
  - `auto_connect` ルールの自動開始は state.yaml の復元後に実行され、既にアクティブなルールはスキップされる

### UC-10: 設定の永続化と変更

- **アクター**: ユーザー
- **概要**: MolePort の設定を変更する
- **CLI**: `moleport config`（設定を表示）
- **TUI**: テーマ変更は `t` キー、言語切替は `l` キーで操作
- **設定項目**:
  - デフォルトの再接続回数上限
  - 再接続のバックオフ設定（初期遅延、最大遅延）
  - KeepAlive 間隔
  - SSH config のパス
  - 自動復元の有効/無効
  - ホストごとの再接続ポリシーオーバーライド（`hosts` セクション）
  - TUI カラーテーマ（ベーステーマ + アクセントカラー）
  - 表示言語（`language`: `ja` / `en`）
  - アップデートチェックの有効/無効（`update_check.enabled`: デフォルト `true`）
  - アップデートチェック間隔（`update_check.interval`: デフォルト `"24h"`）
- **保存先**: `~/.config/moleport/config.yaml`

### UC-11: 転送ルールの削除

- **アクター**: ユーザー
- **概要**: 登録済みの転送ルールを削除する
- **CLI**: `moleport delete <name>`
- **TUI**: 転送一覧で `x` キー
- **基本フロー**:
  1. 転送を停止しルールを削除する
- **補足**:
  - CLI: 確認なしで即座に削除する
  - TUI: 確認ダイアログを表示した後に削除する

### UC-13: SSH 認証時のクレデンシャル入力

- **アクター**: ユーザー / システム（デーモン）
- **概要**: SSH 接続時にパスワード・パスフレーズ・keyboard-interactive 認証が必要な場合に、クライアント経由でユーザーにクレデンシャル入力を求める
- **トリガー**: `moleport connect <host>`、`moleport start <name>`、または TUI からの接続・フォワード開始時に、SSH エージェント・鍵ファイルだけでは認証が完了しない場合
- **基本フロー**:
  1. デーモンが SSH 接続を試行する
  2. パスフレーズ付き秘密鍵の復号失敗、またはサーバーがパスワード/keyboard-interactive 認証を要求
  3. デーモンがクライアントに `credential.request` 通知を送信（認証種別とプロンプト情報を含む）
  4. クライアントがユーザーにプロンプトを表示する
     - CLI: ターミナルの秘密入力（エコーなし）
     - TUI: パスワード入力ダイアログ（マスク表示）
  5. ユーザーがクレデンシャルを入力する
  6. クライアントが `credential.response` でデーモンに応答する
  7. デーモンがクレデンシャルを使って認証を完了する
  8. 認証成功時、接続を確立しクレデンシャルをメモリから消去する
- **代替フロー**:
  - keyboard-interactive で複数チャレンジがある場合: ステップ 3〜6 を繰り返す
  - ユーザーがキャンセルした場合: クライアントが空レスポンスを返し、デーモンが接続を中断する
  - 認証失敗: エラーメッセージを表示し、リトライを促す
  - タイムアウト（30秒以内に応答がない場合）: デーモンが認証を中断しエラーを返す

### UC-14: セッション復元時のクレデンシャル保留

- **アクター**: システム（デーモン）
- **概要**: デーモン起動時のセッション復元・auto_connect で、クレデンシャルが必要なホストを保留状態にする
- **基本フロー**:
  1. デーモン起動時に state.yaml / auto_connect ルールの復元を試みる
  2. SSH エージェント・鍵ファイルのみで認証可能なホストは通常通り接続する
  3. クレデンシャル入力が必要なホストは `pending_auth` 状態にする
  4. クライアント（CLI/TUI）が接続した際に、`pending_auth` のホストがあることをイベント通知する
  5. ユーザーが手動で `connect` を実行すると、UC-13 のフローに進む
- **表示**:
  - CLI `moleport status`: pending_auth ホストの一覧を表示
  - TUI: ホスト一覧で `⏳ Pending Auth` バッジを表示

### UC-15: TUI 初回起動時のテーマ選択

- **アクター**: ユーザー
- **概要**: TUI を初めて起動した際にカラーテーマの選択画面を表示し、好みの配色を設定する
- **前提条件**: config.yaml に `tui.theme` が未設定（初回起動）。言語選択（UC-18）が完了済み
- **TUI**: `moleport tui` 初回起動時に自動表示（言語選択の後）
- **基本フロー**:
  1. TUI 起動時に config.yaml の `tui.theme` を確認する
  2. 未設定の場合、ダッシュボード表示前にテーマ選択画面を表示する
  3. グリッド形式で Dark / Light × 5 アクセントカラーのプリセットを一覧表示する
  4. カーソル移動で選択中のテーマがリアルタイムで TUI 全体に適用される
  5. Enter で確定し、config.yaml に保存してダッシュボードに遷移する
- **代替フロー**:
  - Esc でスキップした場合: デフォルトテーマ（Dark Violet）が適用され、config.yaml に保存される

### UC-16: テーマ設定の変更

- **アクター**: ユーザー
- **概要**: TUI 内のキーバインドからテーマ設定画面を開き、カラーテーマを変更する
- **TUI**: `t` キー
- **基本フロー**:
  1. ダッシュボード上で `t` キーを押す
  2. テーマ選択画面が表示される（UC-15 と同じ画面、現在のテーマが選択状態）
  3. カーソル移動で選択中のテーマがリアルタイムで TUI 全体に適用される
  4. Enter で確定し、config.yaml に保存してダッシュボードに戻る
- **代替フロー**:
  - Esc でキャンセルした場合: 変更前のテーマに戻り、ダッシュボードに戻る

### UC-17: TUI 起動時のバージョン不一致検出

- **アクター**: ユーザー / システム（TUI）
- **概要**: TUI 起動時にデーモンのバージョンを確認し、不一致の場合はデーモン再起動を提案する
- **前提条件**: デーモンが稼働中
- **TUI**: `moleport tui` 起動時に自動実行
- **基本フロー**:
  1. TUI 起動時に `daemon.status` を呼び出し、デーモンのバージョンを取得する
  2. TUI 自身のバージョン（ビルド時に埋め込み）と比較する
  3. バージョンが一致する場合、通常通りダッシュボードを表示する
  4. バージョンが不一致の場合、確認ダイアログを表示する:
     - メッセージ: 「デーモンのバージョン (vX.X.X) が TUI のバージョン (vY.Y.Y) と一致しません。デーモンを再起動しますか？」
     - 選択肢: [はい] / [いいえ]
  5. ユーザーが「はい」を選択した場合:
     a. `daemon.shutdown` でデーモンを停止する
     b. 新しいデーモンプロセスを起動する
     c. 新しいデーモンに再接続する
     d. ダッシュボードを表示する
  6. ユーザーが「いいえ」を選択した場合:
     a. ステータスバーにバージョン不一致の警告を表示する
     b. そのままダッシュボードを表示する（機能制限なし）
- **代替フロー**:
  - デーモンのバージョンが `"dev"` の場合: 開発ビルドとみなしチェックをスキップする
  - TUI のバージョンが `"dev"` の場合: 同上
  - デーモンの再起動に失敗した場合: エラーメッセージをログパネルに表示し、ダッシュボードに遷移する

### UC-18: TUI 初回起動時の言語選択

- **アクター**: ユーザー
- **概要**: TUI を初めて起動した際に言語選択画面を表示し、UI の表示言語を設定する
- **前提条件**: config.yaml に `language` が未設定（初回起動）
- **TUI**: `moleport tui` 初回起動時に自動表示
- **基本フロー**:
  1. TUI 起動時に config.yaml の `language` を確認する
  2. 未設定の場合、ダッシュボード表示前に言語選択画面を表示する
  3. 対応言語をリスト表示する（日本語 / English）
  4. カーソル移動で選択し、Enter で確定する
  5. config.yaml の `language` に保存し、次のセットアップ画面（テーマ選択: UC-15）に遷移する
- **代替フロー**:
  - Esc でスキップした場合: デフォルト言語（en）が適用され、config.yaml に保存される
- **注意**: 言語選択はテーマ選択（UC-15）の前に実施される。言語選択後のテーマ選択画面は選択した言語で表示される

### UC-19: 言語設定の変更

- **アクター**: ユーザー
- **概要**: TUI 内のキーバインドから言語設定画面を開き、表示言語を変更する
- **TUI**: `l` キー
- **基本フロー**:
  1. ダッシュボード上で `l` キーを押す
  2. 言語選択画面が表示される（UC-18 と同じ画面、現在の言語が選択状態）
  3. カーソル移動で選択し、Enter で確定する
  4. config.yaml に保存し、TUI 全体の表示言語を即座に切り替える
- **代替フロー**:
  - Esc でキャンセルした場合: 変更前の言語のまま、ダッシュボードに戻る

### UC-20: CLI の言語自動検出

- **アクター**: ユーザー / システム
- **概要**: CLI コマンド実行時に表示言語を自動的に決定する
- **基本フロー**:
  1. config.yaml の `language` フィールドを確認する
  2. 設定されている場合、その言語を使用する
  3. 未設定の場合、環境変数を以下の優先順で確認する:
     a. `LC_ALL`
     b. `LANG`
  4. 環境変数から言語コードを抽出する（例: `ja_JP.UTF-8` → `ja`）
  5. 対応言語に一致すれば採用、一致しなければデフォルト（`en`）を使用する
- **補足**:
  - CLI は config.yaml の設定を最優先とし、環境変数はフォールバックとして機能する
  - これにより、TUI で言語を選択した後は CLI も同じ言語で表示される

### UC-21: 最新バージョンチェック（CLI）

- **アクター**: ユーザー
- **概要**: `moleport version` 実行時に、現在のバージョン情報に加えて最新バージョンの確認結果を表示する
- **CLI**: `moleport version`
- **前提条件**: `update_check.enabled` が `true`（デフォルト）
- **基本フロー**:
  1. ローカルのバージョン情報を表示する（MolePort vX.X.X (goX.X.X, OS/ARCH)）
  2. デーモンへの接続を試みる
  3. 接続成功した場合、`version.check` IPC メソッドを呼び出す
  4. 最新バージョンの確認結果を表示する:
     - 最新の場合: 「✓ 最新バージョンです」
     - 更新がある場合: 「⬆ v0.2.0 が利用可能です → https://github.com/ousiassllc/moleport/releases/tag/v0.2.0」
     - チェック失敗の場合: 何も表示しない（ローカルバージョンのみ表示）
- **代替フロー**:
  - デーモンが稼働していない場合: ローカルバージョン情報のみを表示し、最新バージョンチェックはスキップする
  - バージョンが `"dev"` の場合: チェックをスキップする
  - `update_check.enabled` が `false` の場合: チェックをスキップする

### UC-22: TUI 起動時の最新バージョン通知

- **アクター**: ユーザー / システム（TUI）
- **概要**: TUI 起動時にデーモンのキャッシュ済み最新バージョン情報を確認し、更新がある場合に確認ダイアログで通知する
- **前提条件**: デーモンが稼働中。`update_check.enabled` が `true`
- **TUI**: `moleport tui` 起動時に自動実行（UC-17 のバージョン不一致チェックの後）
- **基本フロー**:
  1. `version.check` IPC メソッドを呼び出す
  2. 更新がない場合、そのまま通常フローを続行する
  3. 更新がある場合、確認ダイアログを表示する:
     - メッセージ: 「MolePort vX.X.X が利用可能です（現在 vY.Y.Y）」
     - リリース URL を表示
     - 選択肢: [OK]（ダイアログを閉じる）
  4. ダッシュボードの表示を続行する
- **代替フロー**:
  - TUI のバージョンが `"dev"` の場合: チェックをスキップする
  - `update_check.enabled` が `false` の場合: チェックをスキップする
  - `version.check` がエラーの場合: 何も表示せず続行する

### UC-23: デーモンの定期バージョンチェック

- **アクター**: システム（デーモン）
- **概要**: デーモンが定期的に GitHub Releases API から最新バージョン情報を取得し、結果をキャッシュする
- **前提条件**: `update_check.enabled` が `true`（デフォルト）
- **基本フロー**:
  1. デーモン起動後、初回チェックを遅延実行する（起動完了から 10 秒後）
  2. GitHub Releases API (`https://api.github.com/repos/ousiassllc/moleport/releases/latest`) にリクエストを送信する
  3. レスポンスから最新バージョン（`tag_name`）とリリース URL（`html_url`）を取得する
  4. 結果をメモリにキャッシュする（バージョン、URL、チェック日時）
  5. `update_check.interval`（デフォルト: 24 時間）ごとにステップ 2〜4 を繰り返す
  6. クライアントが `version.check` を呼び出した場合、キャッシュ済みの結果を返す
- **代替フロー**:
  - ネットワーク到達不能の場合: エラーをログに記録し、次回チェックまでスキップする。キャッシュ済みの結果があればそれを返す
  - GitHub API レート制限に達した場合: エラーをログに記録し、次回チェックまでスキップする
  - `update_check.enabled` が `false` の場合: 定期チェックを開始しない
  - バージョンが `"dev"` の場合: チェックをスキップする
- **制限事項**:
  - GitHub API 認証は使用しない（パブリックリポジトリ前提、認証なしで 60 リクエスト/時）
  - ディスクへのキャッシュ永続化は行わない（デーモン再起動時に再チェック）

### UC-24: セルフアップデート

- **アクター**: ユーザー
- **概要**: GitHub Releases から最新バイナリをダウンロードし、自動でアップデートする
- **CLI**: `moleport update [--check]`
- **前提条件**: バージョンが `"dev"` でないこと
- **基本フロー**:
  1. GitHub Releases API で最新バージョンを取得する
  2. 現在のバージョンと比較し、新しいバージョンがなければ「最新です」と表示して終了
  3. `--check` 指定時はバージョン情報を表示して終了
  4. OS/アーキテクチャに対応するアセット（tar.gz）をダウンロードする
  5. チェックサムファイル（checksums.txt）をダウンロードし SHA-256 で検証する
  6. デーモンが稼働中の場合はグレースフルに停止する
  7. 現在のバイナリをアトミックリネームで置換する（一時ファイル → rename）
  8. デーモンが稼働していた場合は再起動する
- **代替フロー**:
  - バージョンが `"dev"` の場合: エラー（開発ビルドではアップデート不可）
  - ダウンロードまたはチェックサム検証に失敗した場合: 元のバイナリは変更せずエラーを表示
  - バイナリの書き込み権限がない場合: エラーメッセージで案内

### UC-12: TUI ダッシュボードの起動

- **アクター**: ユーザー
- **概要**: デーモンに接続し、リアルタイムダッシュボードを表示する
- **CLI**: `moleport tui`
- **基本フロー**:
  1. デーモンに IPC 接続する
  2. `daemon.status` でバージョンを確認する（UC-17）
  3. 最新バージョンを確認する（UC-22）
  4. 初回起動時のセットアップを実行する（言語選択: UC-18 → テーマ選択: UC-15）
  5. events.subscribe でイベントストリームを開始
  6. ダッシュボードを表示
  7. TUI 終了時にサブスクリプションを解除し、IPC 切断
  8. デーモンとポートフォワーディングはそのまま継続

## TUI 画面構成

### メイン画面（ダッシュボード）

各セクションは Lip Gloss の Rounded Border で囲み、フォーカス中のパネルはアクセントカラー（バイオレット）のボーダーで強調する。
ステータスバーは背景色付きで画面最下部に固定表示する。

```
  MolePort v0.1.0
╭─ Active Forwards (2) ───────────────────────────────────────────╮
│ > ● [prod] L :8080 ──▸ localhost:80     2h 15m  ↑1.2MB ↓340KB  │
│   ● [prod] L :5432 ──▸ localhost:5432   45m     ↑340KB ↓120KB  │
│                                                                 │
╰─────────────────────────────────────────────────────────────────╯
╭─ SSH Hosts (3) ─────────────────────────────────────────────────╮
│ > ● prod-server    user@192.168.1.10:22     2 fwd               │
│   ○ staging        deploy@10.0.0.5:22       0 fwd               │
│   ○ dev-db         admin@172.16.0.3:5432    0 fwd               │
│                                                                 │
╰─────────────────────────────────────────────────────────────────╯
╭─ Log ───────────────────────────────────────────────────────────╮
│ ✓ prod-server に接続しました                                     │
│ ✓ prod-web フォワードを開始しました                               │
╰─────────────────────────────────────────────────────────────────╯
 3 hosts  2 connected │ 2 forwards  2 active    [Tab] Switch  [?] Help  [q] Quit
```

**レイアウト仕様**:
- **ヘッダー**: アプリ名（アクセントカラー） + バージョン（Muted）を 1 行で表示
- **パネルボーダー**: Rounded Border（`╭╮╰╯`）。フォーカス時はアクセントカラー、非フォーカス時は Dim カラー
- **パネルタイトル**: ボーダー上辺にインラインで表示（例: `─ Active Forwards (2) ─`）
- **選択行**: `> ` プレフィックス + アクセントカラーのテキスト
- **ログパネル**: 固定 3 行。タイトル「Log」付き
- **ステータスバー**: 背景色（BgHighlight `#27272A`）付きの全幅バー。左に統計、右にキーヒント

**変更点（v2 → v3）**:
- 全パネルに Rounded Border を追加（フォーカス状態で色変化）
- 水平区切り線（`───`）を廃止し、パネルボーダーで区切り
- ログパネルにタイトルを追加
- ステータスバーに背景色を追加

**変更点（v1 → v2）**:
- ヘッダーにデーモン接続状態を表示（`⬤ Daemon Connected` / `○ Disconnected`）
- TUI 終了時はデーモン接続を切断するのみ（ポートフォワーディングは継続）

### 言語選択画面

初回起動時または `l` キーで表示される。ダッシュボードの上にオーバーレイとして表示する。

```
╭─ Language / 言語 ─────────────────────────────────────────────╮
│                                                                 │
│   > ● English                                                   │
│     ● 日本語                                                    │
│                                                                 │
│   [↑↓] Select  [Enter] Apply  [Esc] Cancel/Skip                │
│                                                                 │
╰─────────────────────────────────────────────────────────────────╯
```

**レイアウト仕様**:
- **全体**: Rounded Border 付きのオーバーレイパネル。タイトル「Language / 言語」をインライン表示
- **リスト**: 対応言語を `●` 記号 + 言語名で一覧表示。各言語はその言語自体で表示する（英語なら "English"、日本語なら "日本語"）
- **カーソル**: `> ` プレフィックスで現在選択中の言語を示す
- **ナビゲーション**: `↑` / `↓` で言語切り替え
- **操作**: Enter で確定、Esc でキャンセル（初回起動時はスキップ扱いでデフォルト言語 `en` を適用）
- **タイトルの多言語表示**: タイトルは "Language / 言語" のように、全対応言語を併記する。これにより、どの言語のユーザーでも画面の目的を理解できる

### TUI の操作方式

TUI はコマンド入力欄を持たず、キーバインドとウィザードで操作する。TUI から SSH の connect/disconnect は行えない（CLI の `moleport connect` / `moleport disconnect` を使用する）。

デーモンが未稼働の場合は自動的に起動する。

#### フォワード追加ウィザード

SetupPanel（SSH ホスト一覧）でホストを選択し、`Enter` キーを押すとフォワード追加ウィザードが開始される。ウィザードのステップ:

1. **転送種別選択**: Local / Remote / Dynamic から選択
2. **ローカルポート**: ローカル側のポート番号を入力（placeholder: `8080`）
3. **リモートホスト**: リモート側のホスト名を入力（placeholder: `localhost`、Dynamic 転送ではスキップ）
4. **リモートポート**: リモート側のポート番号を入力（placeholder: ローカルポートと同じ値、Dynamic 転送ではスキップ）
5. **ルール名**: 転送ルールの名前を入力（placeholder: `{ホスト名}-{転送種別}-{ローカルポート}`）
6. **確認**: 入力内容を確認し、Enter で追加を実行

- 全テキスト入力ステップ（ステップ 2〜5）で、入力が空のまま Enter を押すと placeholder の値が自動的に採用される
- `Esc` キーでウィザードをキャンセル
- ウィザード完了後、転送ルールが追加され自動的にフォワーディングを開始する

#### TUI で可能な操作

| 操作 | 方法 | 説明 |
|------|------|------|
| フォワード追加 | `Enter`（SetupPanel） | フォワード追加ウィザードを開始 |
| フォワード削除 | `x` キー（転送一覧） | 選択中の転送ルールを削除 |
| テーマ変更 | `t` キー | テーマ選択画面を表示 |
| 言語切替 | `l` キー | 言語切替画面を表示 |
| TUI 終了 | `q` / `Ctrl+C` | TUI を終了（デーモンは継続） |

### 操作フロー

```
moleport daemon start
 │
 ├── config.yaml 読み込み
 ├── state.yaml からセッション復元
 ├── auto_connect ルールの自動接続
 ├── Unix ソケット Listen 開始
 │
 ▼
デーモン常駐（バックグラウンド）
 │
 ├── CLI からの操作:
 │   ├── moleport connect <host>   → SSH 接続 + auto_connect 転送開始
 │   ├── moleport disconnect <host> → SSH 切断 + 全転送停止
 │   ├── moleport add              → 転送ルール追加（対話形式）
 │   ├── moleport delete <name>    → 転送ルール削除
 │   ├── moleport start <name>     → 転送開始
 │   ├── moleport stop <name>      → 転送停止
 │   ├── moleport list             → 一覧表示
 │   ├── moleport status           → 状態サマリー
 │   └── moleport reload           → SSH config 再読み込み
 │
 ├── TUI からの操作:
 │   ├── moleport tui              → ダッシュボード起動
 │   ├── イベントサブスクリプション → リアルタイム更新
 │   └── TUI 終了 → デーモン接続切断のみ（転送は継続）
 │
 ▼
moleport daemon stop
 ├── 全転送の状態を state.yaml に保存
 ├── 全ポートフォワーディング停止
 └── 全 SSH 接続をグレースフルに切断

moleport daemon kill（応答しない場合の緊急手段）
 └── PID ファイルから SIGKILL → PID ファイル削除
```
